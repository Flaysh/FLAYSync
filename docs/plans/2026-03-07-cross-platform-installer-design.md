# FLAYSync Cross-Platform Installer Design

## Goal

Make FLAYSync installable via download & double-click on Mac and Windows. No terminal, no Node.js, no git required for end users.

## Approach: electron-builder

Use electron-builder for packaging, installers, and auto-update. GitHub Actions for CI/CD. Distribute via GitHub Releases, linked from a future website.

## Packaging & Installers

**Mac:**
- `.dmg` with drag-to-Applications layout
- `.zip` for auto-update delivery
- Architectures: x64 + arm64

**Windows:**
- NSIS installer (`.exe`) with standard install wizard
- Installs to Program Files, creates Start Menu + optional desktop shortcut

**App metadata:**
- App name: FLAYSync
- App ID: com.flaysh.flaysync
- Icons: `.icns` (Mac) and `.ico` (Windows) generated from `assets/flaysh-logo.png`

**No code signing** for now. Users bypass Gatekeeper / SmartScreen on first install.

## Auto-Update

- `electron-updater` (bundled with electron-builder)
- On launch: silently checks GitHub Releases for newer version
- If found: shows UI notification "Update available (vX.Y.Z). Restart to update?"
- User clicks to update, downloads in background, restarts
- Update feed: `latest.yml` / `latest-mac.yml` auto-generated by electron-builder on GitHub Releases
- Works without code signing

## CI/CD (GitHub Actions)

**Trigger:** Push a git tag matching `v*` (e.g., `v0.5.0`), or manual dispatch.

**Build matrix:**
- `macos-latest`: builds `.dmg` + `.zip`
- `windows-latest`: builds NSIS `.exe`

**Steps per platform:**
1. Checkout code
2. Setup Node.js 18 + pnpm
3. `pnpm install` (includes native module rebuild via postinstall)
4. `pnpm test`
5. `npx electron-builder` â€” package and build installer
6. Upload artifacts to draft GitHub Release

**Release flow:**
1. Bump version in `package.json`
2. `git tag v0.5.0 && git push --tags`
3. GitHub Actions builds both platforms in parallel
4. Draft Release appears with installers attached
5. Review, add release notes, publish

**Native module (`abletonlink-addon`):** Compiled from source on each CI runner via `@electron/rebuild`. If Windows CI fails, fall back to prebuild approach.

## Decisions

- Package manager: pnpm
- No code signing (add later)
- Auto-update with user prompt (not silent)
- GitHub Releases as update backend
- electron-builder over electron-forge (less migration, better auto-update)
